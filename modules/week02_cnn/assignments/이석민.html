🎭 Week 06: 세그멘테이션 & SAM 실습 결과

6.1 학습한 핵심 개념

  • Semantic Segmentation: 픽셀 단위 의미론적 분할
  • Instance Segmentation: 개별 객체 인스턴스 분할
  • Panoptic Segmentation: Semantic + Instance 통합
  • U-Net 아키텍처: Encoder-Decoder 구조의 세그멘테이션 모델
  • SAM (Segment Anything Model): Meta의 범용 세그멘테이션 모델

U-Net의 주요 특징:

  • U자 형태 구조: 압축-확장 경로로 구성
  • Skip Connection: 고해상도 정보 보존
  • 의료 영상 분할: 초기 목적, 현재는 범용적 사용
  • 픽셀 레벨 예측: 각 픽셀의 클래스 분류

6.2 실습 성과

  • 다양한 세그멘테이션 기법 비교 분석
  • U-Net 구조와 동작 원리 이해
  • SAM 모델의 프롬프트 기반 세그멘테이션 체험
  • 실시간 세그멘테이션 처리 파이프라인 구현

🏃 Week 07: 액션 인식 실습 결과

7.1 학습한 핵심 개념

  • 3D CNN: 시공간 특징 추출
  • Two-Stream Networks: RGB + Optical Flow
  • Video Transformer: VideoMAE, TimeSformer
  • MediaPipe: 실시간 포즈 검출
  • Optical Flow: 움직임 벡터 계산

MediaPipe의 강력한 기능들:

  • Pose Detection: 33개 랜드마크 실시간 검출
  • Hand Tracking: 21개 손 랜드마크 추적
  • Face Mesh: 468개 얼굴 키포인트
  • Holistic: 포즈 + 손 + 얼굴 통합 분석

7.2 실습 성과

  • 실시간 포즈 검출 및 분석
  • 운동 동작 자동 카운팅 시스템 구현
  • YouTube 영상 다운로드 및 분석 파이프라인
  • 다양한 액션 인식 모델 비교 분석

😊 Week 08: 감정 인식 실습 결과

8.1 학습한 핵심 개념

  • 7가지 기본 감정: Happy, Sad, Angry, Fear, Surprise, Disgust, Neutral
  • VAD 모델: Valence-Arousal-Dominance 3차원 감정 공간
  • 멀티모달 분석: 이미지 + 텍스트 통합 감정 인식
  • 시계열 감정 추적: 감정 변화 패턴 분석
  • 3-tier Fallback: Gemini → OpenAI → Simulation

🚨 해결한 실제 문제: API Rate Limiting

에러: "429 You exceeded your current quota" (Gemini API 무료 티어 제한)

제한: 분당 2회 요청만 허용

💡 구현한 해결책:

  • RateLimiter 클래스: 요청 시간 추적 및 자동 대기
  • 스마트 재시도: 에러 메시지에서 대기 시간 자동 추출
  • 지수 백오프: 점진적 대기 시간 증가
  • Fallback 전략: API 실패 시 시뮬레이션 모드 자동 전환

8.2 VAD 3차원 감정 모델

  • Valence (긍정도): 긍정적 ↔ 부정적 (-1.0 ~ 1.0)
  • Arousal (각성도): 차분함 ↔ 흥분 (-1.0 ~ 1.0)
  • Dominance (지배도): 복종 ↔ 지배 (-1.0 ~ 1.0)

8.3 실습 성과

  • Google Gemini API를 활용한 실시간 감정 분석
  • API Rate Limiting 문제 실무적 해결 경험
  • 멀티모달 분석을 통한 더 정확한 감정 인식
  • VAD 3차원 모델로 감정의 세밀한 분류
  • 시계열 감정 변화 추적 시스템 구현

가장 값진 경험: Google Gemini API Rate Limiting 해결

  • 문제상황: 무료 티어 API 제한 (분당 2회 요청)
  • 해결방법: RateLimiter 클래스 구현 및 스마트 재시도 로직
  • 학습효과: 실제 상용 API의 제약사항 이해 및 대응 능력 향상
